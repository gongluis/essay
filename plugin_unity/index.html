开发unity插件的初衷

在mob负责的安卓端短信sdk及一键登录秒验sdk需要做全平台适配，其中就涉及到主流游戏平台之一的unity，公司的全平台适配方案恰好又是插件开发，于是用一周自学unity,然后又花了一周做出了秒验sdk插件。本篇目的是为了记录开发过程中的一些流程，属于unity3D入门基础内容。

基本理解

unity插件开发需要开发的部分主要有两个部分：
1. jar/aar(简单包装原sdk) 2. C#层负责桥接的bridge

详细描述整个过程： 

1. C#通过一些开发是可以直接接入jar或者aar的，但是为了接入方的方便我们应当开发好桥接层后提供出去，
2. 正常第一步需要新建一个library工程来对原生的sdk进行一层包装，
3. 包装的第一步，引入unity的calss.jar放到lib中，
然后在项目中就可以调用unity的UiHandler.sendMessage对unity层进行发消息通信，回调结果。
4. 如果没有异步的需求的话可以不用，直接采用同步回调。
5. 最后将以上打出的jar包引入到unity的（Assets-Plugins-Android），便可以调用其中方法。

C#调用android(简单描述)

在unity中创建的AndroidJavaClass对象可以调用一些静态方法，创建的AndroidJavaObject用来调用对象中的方法。

一、拿到AndroidJavaClass或者AndroidJavaObject
  1. AndroidJavaClass adClass = new AndroidJavaClass("com.secverify.unity3d.SecVerifyUtils");
  2. AndroidJavaObject adObject = new AndroidJavaObject("com.secverify.unity3d.SecVerifyUtils");
二、通过拿到的class或者object调用Android

adObject.Call("init", appKey, appSecret); init为方法名，后面两个appKey和appSecret为参数

//android中的方法：
public class SMSSDKUtils  {
    public void init(String appKey, String )
    {

    }
}

如果有非异步的返回值：
String str = adObject.call("init", appKey, appSecret);

Android调用C#（简单描述）

简单可分为两步： 1. android项目中引入unity.jar取自unitysdk

 UnityPlayer.UnitySendMessage("Main Camera", "_callback", formatStr);

 第一个参数是 脚本挂载的物体名称
 第二个参数是该脚本中的准备接收回调的方法名
 第三个参数是要传递过去的参数（String类型）
  1. 在unity的脚本中定义该方法，取出值，复杂的可用map或者json来传值
 private void _callback(string callbackData) {
    ...

UnityHub激活新许可证报错处理

选择个人版，用于初学者和业余爱好者
如果激活新许可证报网络错误
这可能是由于你之前装过其他版本的Unity导致的，而不是网络原因！

进入 C:\ProgramData 找到 Unity 文件夹，
将整个 Unity 文件夹删掉，
然后重新启动Unity Hub 激活新许可证 即可！
